# Go 语言学习计划

## 目标

本计划用于指导本仓库的 Go 语言学习过程。

学习方式以手敲代码为主，目标不是快速堆框架，而是逐步熟悉 Go 的语言特性、标准库、工程习惯和后端开发方式。参考资料可以提供方向，但本项目会做取舍：先学会写清楚、跑得通、容易测试的 Go 代码，再进入 Web、数据库、缓存、消息队列和微服务实践。

## 学习原则

- 每个阶段只解决一组核心问题，不贪多。
- 每个主题都要有可手敲、可运行、可复盘的示例。
- 优先使用 Go 标准库，必要时再引入第三方库。
- 不急着套框架，先理解 Go 原生写法和工具链。
- 每个阶段结束前，至少完成一次运行或测试验证。

## 阶段一：环境、工具链与第一段程序

对应目录：`study_1`

学习重点：

- Go 项目的基本结构
- `go mod` 的作用
- `go run`、`go build`、`go test`、`gofmt`
- `package main`、`func main`
- 基础输出和简单函数

练习目标：

- 能独立创建一个 Go module。
- 能运行第一段 Go 程序。
- 能理解 Go 文件、包和模块之间的基本关系。

## 阶段二：基础语法与控制流程

对应目录：`study_2`

学习重点：

- 变量、常量、零值
- 基本数据类型
- 运算符
- `if`、`for`、`switch`
- 函数、多返回值、`defer`

练习目标：

- 编写若干小函数处理字符串、数字和条件分支。
- 熟悉 Go 中没有 `while` 的写法。
- 理解多返回值如何服务于错误处理。

## 阶段三：数组、切片、映射与字符串

对应目录：`study_3`

学习重点：

- 数组与切片
- `append`、切片截取、长度和容量
- `map` 的增删改查
- 字符串、`rune`、字节
- 常用标准库：`strings`、`strconv`

练习目标：

- 写一个简单的单词统计程序。
- 写一个切片去重或筛选函数。
- 理解切片共享底层数组带来的影响。

## 阶段四：结构体、方法与组合

对应目录：`study_4`

学习重点：

- 结构体定义和初始化
- 方法和接收者
- 值接收者与指针接收者
- 结构体嵌入
- JSON 序列化与字段标签

练习目标：

- 用结构体实现一个简单的用户或订单模型。
- 为结构体编写方法。
- 完成结构体与 JSON 的相互转换。

## 阶段五：接口与错误处理

对应目录：`study_5`

学习重点：

- 接口的隐式实现
- 小接口的设计习惯
- `error` 接口
- `errors`、`fmt.Errorf`
- `panic` 和 `recover` 的边界

练习目标：

- 定义一个小接口并编写两个实现。
- 编写带错误返回的函数。
- 理解正常业务错误不应依赖 `panic`。

## 阶段六：包、模块与测试

对应目录：`study_6`

学习重点：

- 包的拆分
- 导出与非导出标识符
- `go test`
- 表格驱动测试
- 基准测试入门

练习目标：

- 将一个小程序拆成多个包。
- 为核心函数编写表格驱动测试。
- 能读懂测试失败时的输出。

## 阶段七：并发基础

对应目录：`study_7`

学习重点：

- goroutine
- channel
- `select`
- `sync.WaitGroup`
- `sync.Mutex`
- `context` 入门

练习目标：

- 编写并发任务执行示例。
- 使用 channel 汇总结果。
- 使用 `context` 控制取消或超时。

## 阶段八：常用标准库与命令行小工具

对应目录：`study_8`

学习重点：

- 文件读写
- 时间处理
- 日志
- JSON
- `flag`
- `net/http` 客户端基础

练习目标：

- 写一个读取文件并统计内容的小工具。
- 写一个命令行参数驱动的小程序。
- 请求一个 HTTP 接口并解析 JSON 响应。

## 阶段九：HTTP 服务入门

对应目录：`study_9`

学习重点：

- `net/http` 服务端
- 路由基础
- 请求参数解析
- JSON 响应
- 中间件思想
- 优雅关闭入门

练习目标：

- 使用标准库写一个简单 REST API。
- 实现统一 JSON 响应。
- 添加简单日志中间件。

## 阶段十：数据库、缓存与消息队列

对应目录：`study_10`

学习重点：

- `database/sql`
- MySQL 基础访问
- Redis 基础访问
- 消息队列基本生产与消费
- 配置管理和连接生命周期

练习目标：

- 完成一个小型 CRUD 示例。
- 使用 Redis 做一次缓存读写。
- 完成一个最小消息生产和消费示例。

## 阶段十一：小型后端项目实践

对应目录：`study_11`

学习重点：

- 项目结构组织
- 配置、日志、错误处理
- HTTP API
- MySQL 与 Redis 集成
- 测试与可维护性

练习目标：

- 完成一个小型后端服务。
- 能清楚说明每个包的职责。
- 能通过测试或手动请求验证核心流程。

## 暂缓深入的内容

以下内容不作为前期重点，等基础稳定后再进入：

- Web 框架源码
- 大型微服务框架
- gRPC 网关和复杂服务治理
- 可观测性体系
- ORM 深度使用
- 性能调优和底层内存布局
- 复杂依赖注入工具

这些内容并不是不重要，而是不适合在刚开始学习 Go 时过早展开。

## 每阶段交付物

每个阶段尽量包含：

- 可运行的手敲代码
- 简短学习笔记
- 至少一个练习任务
- 必要时沉淀到 `知识问答` 的小问题
- 一次运行或测试验证记录
