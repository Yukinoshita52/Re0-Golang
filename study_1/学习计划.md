# 第一阶段学习计划：环境、工具链与第一段程序

## 阶段目标

完成这一阶段后，你应该能看懂一个最小 Go 项目的组成，知道如何运行、格式化和简单验证代码，并能手敲一个很小的 Go 程序。

本阶段只关注入门必需内容，不展开复杂语法。

## 准备确认

开始前确认：

- GoLand 能打开 `study_1` 目录。
- 目录中存在 `go.mod` 和 `main.go`。
- 终端可以在 `study_1` 目录下执行 Go 命令。

## 执行步骤

### 1. 观察项目结构

打开 `study_1`，先只看两个文件：

- `go.mod`
- `main.go`

要确认的问题：
- `go.mod` 里模块名是什么？
```text
study_1
```
- `main.go` 中包名是什么？
```text
main
```
- 程序入口函数叫什么？
```text
main
```

完成标志：

- 能用自己的话说出 `go.mod` 和 `main.go` 分别负责什么。

### 2. 运行第一段程序

在 `study_1` 目录下执行：

```bash
go run .
```

完成标志：

- 终端能看到程序输出。
- 如果运行失败，先记录报错信息，再定位是目录、Go 环境还是代码问题。

### 3. 手敲一次最小程序

修改 `main.go`，保留最小结构：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, Go")
}
```

然后再次执行：

```bash
go run .
```

完成标志：

- 能成功输出 `Hello, Go`。
- 能指出 `package main`、`import`、`func main` 的位置。

### 4. 体验格式化

故意把缩进打乱，然后执行：

```bash
gofmt -w main.go
```

完成标志：

- 看到代码被自动整理。
- 理解 Go 代码格式一般交给 `gofmt`，不用手动纠结风格。

### 5. 添加一个简单函数

在 `main.go` 中新增一个函数，例如：

```go
func greeting(name string) string {
	return "Hello, " + name
}
```

然后在 `main` 中调用它：

```go
fmt.Println(greeting("Go"))
```

完成标志：

- 程序能运行。
- 能看出函数参数、返回值和函数调用的位置。

### 6. 做一次最小测试体验

新增文件 `main_test.go`，手敲：

```go
package main

import "testing"

func TestGreeting(t *testing.T) {
	got := greeting("Go")
	want := "Hello, Go"

	if got != want {
		t.Fatalf("got %q, want %q", got, want)
	}
}
```

执行：

```bash
go test
```

完成标志：

- 测试通过。
- 能理解测试文件以 `_test.go` 结尾。
- 能看懂 `got` 和 `want` 分别表示实际结果和期望结果。

## 本阶段不展开

这些内容先知道名字即可，不在第一阶段深入：

- 指针
- 接口
- goroutine
- channel
- Web 服务
- 数据库连接

## 复盘清单

完成后，用自己的话回答：

- `go.mod` 是做什么的？

```text
做依赖管理、go sdk版本声明等
```

- 为什么 Go 程序入口是 `func main()`？

```text
go语言固定的规则，在`package main`中，`func main()`是程序运行入口。
ps：如果没有package main，那就不是一个可运行的程序。但是可以没有package main
那就是作为库函数给其他地方使用
```

- `go run .` 做了什么？

```text
编译当前Go包，如果是可执行程序，则运行入口函数func main()
```

- `gofmt` 解决了什么问题？

```text
格式化代码
```

- `go test` 默认会识别什么样的测试文件？

```text
以_test结尾的.go文件。测试函数通常形如`func TestXxx(t *testing.T)`
```

如果其中某个问题说不清楚，可以先提问。需要长期保留的问题，再沉淀到 `知识问答` 目录。
