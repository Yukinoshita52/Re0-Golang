# Re0-Golang

这是一个从零到一学习 Go 语言的个人学习仓库。

本仓库的学习方式以手敲代码为主。学习者已有后端开发经验，因此学习重点会放在 Go 语言自身的语法、标准库、工程习惯和后端实践上。

## 学习原则

- 每个学习阶段都尽量通过手敲代码完成。
- 优先学习 Go 标准库和语言特性，再逐步引入生态工具。
- 示例代码保持小而清晰，便于理解、运行和复盘。
- 重点学习 Go 的惯用写法，而不是照搬其他语言的开发习惯。

## 仓库结构

```text
.
├── AGENT.md        # AI 协作规则
├── README.md       # 项目说明
├── study_1/        # 第 1 阶段学习项目，由 GoLand 创建和维护
└── 知识问答/       # 学习过程中沉淀的问答笔记
```

`study_x` 目录用于按阶段存放代码练习和学习笔记。

## 阶段规划

推荐学习路线如下：

1. Go 安装、模块初始化、`go run`、`go test`、`go fmt`、`go mod`
2. 基础语法、变量、常量、运算符、流程控制
3. 函数、多返回值、命名返回值、`defer`
4. 指针和值语义
5. 数组、切片、映射、字符串
6. 结构体、方法、嵌入、组合
7. 接口和依赖边界
8. 错误处理
9. 包与模块
10. 测试与基准测试
11. 并发：goroutine、channel、同步原语
12. 后端开发常用标准库
13. HTTP API
14. MySQL、Redis、消息队列集成
15. 生产风格项目结构与微服务实践

这份路线会随着实际学习进度调整。

## 知识问答

`知识问答` 目录用于保存学习过程中值得回看的小概念、小区别和小术语。

## 长期目标

完成本仓库学习后，应能够：

- 阅读和编写较地道的 Go 代码。
- 理解 Go 与 Java 在设计哲学上的差异。
- 编写小型命令行程序。
- 使用标准库和合适的生态工具构建 HTTP 服务。
- 在 Go 中使用 MySQL、Redis 和消息队列。
- 编写测试并理解 Go 并发模型。
- 以 Go 友好的方式组织一个小型后端服务。
